# C4 Architecture Documentation (RU/EN)

## RU

### Уровень 1: System Context

```mermaid
flowchart LR
    User["Пользователь (Browser)"] --> FE["1CBAS Frontend (Vercel, React/Vite)"]
    FE --> BE["1CBAS Backend API (Vercel Serverless)"]
    BE --> MEM["Runtime Storage (in-memory)"]
    Dev["Разработчик/QA"] --> E2E["Playwright E2E"]
    E2E --> FE
    E2E --> BE
```

Описание:
- Пользователь работает с веб-интерфейсом.
- Frontend вызывает backend API.
- Backend в текущем Vercel serverless варианте использует runtime storage (непостоянное хранилище).
- QA/разработчик проверяет сценарии через Playwright.

### Уровень 2: Containers

```mermaid
flowchart TB
    subgraph Frontend["Container: Frontend"]
        FE1["React App"]
        FE2["Zustand Store"]
        FE3["Axios Client"]
        FE4["Socket.IO Client"]
    end

    subgraph Backend["Container: Backend API"]
        BE1["Vercel Function api/index.js"]
        BE2["REST Endpoints /api/*"]
    end

    subgraph LocalMode["Container: Local Backend (dev)"]
        L1["Express app server/server.js"]
        L2["SQLite DB server/1cremix.db"]
    end

    Frontend --> Backend
    Backend --> Frontend
    L1 --> L2
```

Описание:
- Production split-hosting: frontend и backend развернуты отдельно.
- Локально backend может работать как Express + SQLite.

### Уровень 3: Components

#### Frontend Components

```mermaid
flowchart LR
    App["App.jsx + Router"] --> Pages["Pages/*"]
    Pages --> Store["documentStore (Zustand)"]
    Pages --> Notify["NotificationContext"]
    Store --> API["runtime.js -> VITE_API_BASE_URL"]
    Pages --> API
```

Ключевые компоненты frontend:
- `client/src/App.jsx`: маршрутизация.
- `client/src/pages/*`: экраны UI (Dashboard, Documents, Editor, Reports и др.).
- `client/src/store/documentStore.js`: загрузка/фильтрация документов, сокеты.
- `client/src/config/runtime.js`: runtime-конфиг API/Socket.

#### Backend Components

```mermaid
flowchart LR
    Entry["api/index.js"] --> Products["GET /api/products"]
    Entry --> Documents["CRUD /api/documents"]
    Entry --> Posting["POST /api/documents/:id/post"]
    Entry --> Reports["/api/reports/*"]
```

Ключевые компоненты backend:
- `api/index.js`: serverless entrypoint, маршруты API, CORS, обработка JSON.
- `server/server.js` + `server/controllers/*` + `server/repositories/*`: локальный полнофункциональный backend.
